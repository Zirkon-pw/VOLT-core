## 3. Ядро редактора — разделение по функциям

Каждая функция редактора — отдельный модуль в `kernel/editor/`:

| Модуль | Ответственность | Файлы |
|--------|----------------|-------|
| `core/` | Создание экземпляра Tiptap, конфигурация редактора, жизненный цикл | `EditorEngine.ts`, `EditorInstance.ts`, `EditorConfig.ts` |
| `extensions/` | Расширения Tiptap (slash-команда, блок кода, таблица, математика, embed, поиск по файлу) | По подпапке на каждое расширение |
| `serialization/` | Конвертация Markdown ↔ Prosemirror | `MarkdownParser.ts`, `MarkdownSerializer.ts` |
| `auto-save/` | Тайминг автосохранения, отслеживание dirty-состояния, регистрация обработчиков сохранения | `AutoSaveService.ts`, `DirtyTracker.ts` |
| `image-handling/` | Drag-drop изображений, вставка, разрешение, конвертация base64 | `ImageDropHandler.ts`, `ImagePasteHandler.ts`, `ImageResolver.ts` |
| `sessions/` | Управление сессиями редактора, якоря, сессии плагинов | `EditorSessionManager.ts`, `EditorSession.ts`, `AnchorManager.ts` |
| `toc/` | Генерация оглавления из заголовков | `TocGenerator.ts` |
| `ui/` | UI-компоненты панели редактора | `EditorPanel.tsx`, `MarkdownEditorSurface.tsx` |

## 4. Система плагинов ядра

| Модуль | Ответственность |
|--------|----------------|
| `loader/` | Загрузка исходного кода плагинов, песочница выполнения, валидация манифеста |
| `api/` | Создание песочница API для каждого плагина, типы API v5, контроль разрешений |
| `events/` | Шина событий с отслеживаемыми подписками, безопасное межплагинное взаимодействие |
| `lifecycle/` | Менеджер жизненного цикла: вызов onLoad, onUnload, onSettingsChange, очистка |
| `registry/` | Реестр UI-регистраций плагинов, реестр настроек плагинов |
| `ui/` | UI-компоненты системы плагинов: диалог подсказок, диалог разрешений, статус задач |

## 5. Менеджер рабочих пространств

| Модуль | Ответственность |
|--------|----------------|
| `core/` | Жизненный цикл workspace, сущность Workspace, Zustand store |
| `panes/` | Конфигурация раскладки панелей (primary/secondary split), UI разделителя, Zustand store |
| `tabs/` | Жизненный цикл вкладок (файлы + плагины), dirty-состояние, UI панели вкладок |
| `ui/` | Основная оболочка WorkspaceShell, панель инструментов |

## 6. Встроенные плагины

Каждый встроенный плагин имеет:
- `manifest.json` — манифест плагина (id, версия, зависимости, разрешения)
- `*Plugin.ts` — точка входа с хуками жизненного цикла (`onLoad`, `onUnload`, `onSettingsChange`)
- `*Service.ts` — сервис данных (вызовы API бэкенда, бизнес-логика)
- `*Store.ts` — Zustand store (состояние плагина)
- `*View.tsx` / `*.tsx` — UI-компоненты
- Дополнительные модули по необходимости
