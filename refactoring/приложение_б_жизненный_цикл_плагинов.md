# Приложение Б — Жизненный цикл плагина v5

## 1. Последовательность событий

```
1. Пользователь открывает рабочее пространство
2. WorkspaceManager генерирует событие 'workspace:opened'
3. PluginLifecycleManager обнаруживает включённые плагины
4. Для каждого плагина:
   a. Загружает исходный код main.js через StorageHandler
   b. Создаёт песочница API контекст (PluginApiFactory)
   c. Выполняет onLoad(api) — плагин регистрирует UI, команды и т.д.
   d. Генерирует 'workspace:opened' плагину через onWorkspaceOpen hook
5. Плагин рендерит зарегистрированные UI-компоненты (сайдбар, тулбар и т.д.)
6. Пользователь взаимодействует с плагином
7. При изменении настроек → вызывается onSettingsChange(event)
8. Пользователь закрывает рабочее пространство:
   a. PluginLifecycleManager вызывает onUnload(api) для каждого плагина
   b. Выполняется очистка плагина
   c. Все регистрации удаляются
   d. Подписки на события очищаются
   e. Процессы завершаются
```

## 2. Детализация шагов

### 2.1 Загрузка плагина

```
PluginLoader.load(pluginId):
  1. Читает manifest.json из ~/.volt/plugins/<pluginId>/
  2. Валидирует манифест (PluginValidator)
  3. Проверяет, включён ли плагин (plugin-state.json)
  4. Читает main.js из ~/.volt/plugins/<pluginId>/
  5. Возвращает исходный код
```

### 2.2 Создание песочницы

```
PluginApiFactory.create(pluginId, manifest):
  1. Создаёт объект api с namespaces: fs, workspace, search, assets, process, ui, editor, events, storage, settings, plugins
  2. Каждый метод обёрнут в PermissionChecker
  3. PermissionChecker проверяет разрешения из манифеста
  4. Если разрешение отсутствует — выбрасывает ошибку
  5. Пути оборачиваются в branded-типы (FilePath, DirPath)
```

### 2.3 Выполнение onLoad

```
PluginLifecycleManager.invokeOnLoad(api, pluginSource):
  1. Выполняет pluginSource через new Function('api', source)
  2. Получает модуль с экспортированными функциями
  3. Вызывает module.onLoad(api)
  4. Если onLoad возвращает функцию — сохраняет как cleanup-функцию
  5. Если onLoad выбрасывает ошибку — логирует, прекращает загрузку плагина
```

### 2.4 Регистрация UI

```
Плагин вызывает:
  api.ui.registerSidebarPanel({ id, title, render })
  api.ui.registerCommand({ id, name, icon, hotkey, callback })
  api.ui.registerPage({ id, title, mode, render, cleanup })
  и т.д.

PluginRegistry сохраняет регистрации:
  sidebarPanels: Map<id, SidebarPanelConfig>
  commands: Map<id, CommandConfig>
  pages: Map<id, PageConfig>
  fileViewers: Map<id, FileViewerConfig>
  slashCommands: Map<id, SlashCommandConfig>
  contextMenuItems: Map<id, ContextMenuItemConfig>
  toolbarButtons: Map<id, ToolbarButtonConfig>
  sidebarButtons: Map<id, SidebarButtonConfig>
  settingsPages: Map<id, SettingsPageConfig>
```

### 2.5 Генерация workspace:opened

```
PluginLifecycleManager.invokeOnWorkspaceOpen(api, workspace):
  1. Если плагин экспортирует onWorkspaceOpen — вызывает
  2. Передаёт { voltId, voltPath, rootPath }
  3. Плагин может инициализировать данные workspace
```

### 2.6 Изменение настроек

```
При изменении любой настройки:
  1. SettingsStore обновляет значение
  2. PluginSettingsStore генерирует событие
  3. PluginLifecycleManager вызывает onSettingsChange(api, event) для каждого плагина
  4. event: { key, value, values }
```

### 2.7 Выгрузка плагина

```
PluginLifecycleManager.unload(pluginId):
  1. Вызывает onUnload(api) если экспортирован
  2. Вызывает cleanup-функцию из onLoad если есть
  3. Для каждой регистрации:
     a. Если у регистрации есть cleanup — вызывает
     b. Удаляет регистрацию из PluginRegistry
  4. Очищает все подписки на события из PluginEventBus
  5. Завершает все процессы плагина
  6. Очищает вкладки плагина из TabStore
  7. Удаляет плагин из PluginLoader
```

## 3. Обработка ошибок

| Сценарий | Поведение |
|----------|-----------|
| Ошибка в `onLoad()` | Логирование, прекращение загрузки, показ ошибки пользователю |
| Ошибка в `onUnload()` | Логирование, продолжение выгрузки (force cleanup) |
| Ошибка в `onSettingsChange()` | Логирование, продолжение работы |
| Ошибка в `onWorkspaceOpen()` | Логирование, продолжение работы |
| Ошибка в callback регистрации UI | Логирование, удаление регистрации |
| Ошибка в cleanup-функции | Логирование, force удаление регистрации |
| Ошибка в PermissionChecker | Блокировка вызова, логирование, ошибка плагину |

## 4. Состояния плагина

```
discovered → enabled → loading → loaded → active → unloading → unloaded
     ↓          ↓          ↓        ↓        ↓          ↓          ↓
  обнаружен  включён   загружается  загружен  активен   выгружается  выгружен
```

| Состояние | Описание |
|-----------|----------|
| `discovered` | Плагин обнаружен в `~/.volt/plugins/`, манифест прочитан |
| `enabled` | Пользователь включил плагин (запись в `plugin-state.json`) |
| `loading` | Загрузка исходного кода, валидация, создание песочницы |
| `loaded` | `onLoad()` выполнен успешно, UI зарегистрирован |
| `active` | Плагин работает, обрабатывает события, рендерит UI |
| `unloading` | Вызван `onUnload()`, выполняется очистка |
| `unloaded` | Плагин полностью выгружен, все ресурсы освобождены |
