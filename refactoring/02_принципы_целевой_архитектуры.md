# Принципы целевой архитектуры

## 1. Бэкенд (Go) — «Слой ОС»

Бэкенд отвечает **только** за:

- **Файловые операции** — чтение, запись, список, создание, удаление, переименование файлов/директорий
- **Управление процессами** — запуск, мониторинг, завершение ОС-процессов (для плагинов)
- **Системные диалоги** — открытие диалогов выбора файлов/папок
- **Wails-мост** — предоставление вышеперечисленного как вызываемых через Wails методов
- **KV-персистентность** — сырое key-value хранилище для настроек фронтенда, данных плагинов, списка vault'ов

**Всё остальное перемещается на фронтенд** (как функции ядра или плагины).

## 2. Фронтенд — Минимальное ядро + Плагины

Ядро фронтенда **минимально**:

- **Движок редактора** — основа Tiptap, сериализация markdown, управление сессиями редактора
- **Система плагинов** — загрузка плагинов, песочница API, шина событий, управление жизненным циклом
- **Менеджер рабочих пространств** — жизненный цикл workspace, раскладка панелей, управление вкладками
- **Оболочка приложения** — маршрутизация, провайдер тем, провайдер i18n, граница ошибок

**Всё остальное — плагин** (встроенный или установленный пользователем):

- Файловое дерево (сайдбар)
- Поиск и палитра команд
- Сервис обработки изображений
- Сервис link preview
- Управление хранилищами (vault manager)
- Страницы настроек
- Хлебные крошки
- Вкладки файлов

## 3. Плагин API v5

- **Хуки жизненного цикла**: `onLoad()`, `onUnload()`, `onSettingsChange()`, `onWorkspaceOpen()`
- **Строгие типы TypeScript**: дженерики, branded-типы, улучшенное автодополнение IDE
- **Межплагинное взаимодействие**: безопасная передача сообщений между плагинами
- **Кастомный UI настроек**: плагины могут опционально рендерить собственный UI настроек

## 4. Дизайн-система

- **Строгие дизайн-токены**: цвета, отступы, типографика, border-radius, тени — всё токенизировано
- **Унифицированные UI-компоненты**: все 12+ компонентов следуют одному дизайн-языку на основе токенов
- **Линт-контроль**: правила ESLint + stylelint предотвращают захардкоженные значения
- **Паритет тем**: светлая и тёмная темы используют идентичную структуру токенов

## 5. Разделение ответственности

| Слой | Ответственность | Примеры |
|------|----------------|---------|
| **Backend domain** | Доменные сущности, value objects, интерфейсы репозиториев, ошибки домена | `FileEntry`, `FileRepository`, `ProcessHandle` |
| **Backend application** | Application-сервисы, use-case'ы, DTO | `FileApplicationService.Read()`, `ProcessApplicationService.Start()` |
| **Backend infrastructure** | Конкретные реализации: ОС-файлы, os/exec, Wails runtime | `FileRepository` (os), `ProcessRepository` (os/exec) |
| **Backend interfaces** | Wails-обработчики — поверхность API для фронтенда | `FileHandler`, `ProcessHandler`, `DialogHandler`, `StorageHandler` |
| **Frontend kernel** | Ядро: редактор, плагины, workspace, навигация | `EditorEngine`, `PluginLoader`, `WorkspaceManager` |
| **Frontend plugins** | Встроенные плагины: файловое дерево, поиск, настройки и т.д. | `FileTreePlugin`, `SearchPlugin`, `SettingsPlugin` |
| **Frontend shared** | Общие утилиты: API-клиенты, UI-кит, дизайн-токены, i18n | `fileApi.ts`, `Button`, `tokens.scss` |
