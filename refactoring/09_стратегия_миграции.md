# Стратегия миграции — Big Bang

## 1. Обзор

Миграция выполняется **единым подходом** (big bang) — все изменения разрабатываются в отдельной ветке, тестируются и мержатся только когда всё готово. Инкрементальный мерж в основную ветку не производится.

## 2. Фазы

### Фаза 1: Реструктуризация бэкенда

**Цель:** Бэкенд компилируется с новой архитектурой, генерируются новые Wails-биндинги.

| Шаг | Действие |
|-----|----------|
| 1.1 | Создать директорию `backend/` с новой структурой (domain/, application/, infrastructure/, interfaces/, bootstrap/) |
| 1.2 | Переместить `core/file/` → `backend/domain/file/` |
| 1.3 | Создать `backend/domain/process/`, `backend/domain/dialog/`, `backend/domain/storage/` |
| 1.4 | Переместить `commands/file/` → `backend/application/file/service.go` (объединить команды в сервис) |
| 1.5 | Создать `backend/application/process/`, `backend/application/dialog/`, `backend/application/storage/` |
| 1.6 | Переместить `infrastructure/` → `backend/infrastructure/` (переименовать persistence → storage) |
| 1.7 | Переместить `interfaces/wailshandler/` → `backend/interfaces/` (сократить с 9 до 4 обработчиков) |
| 1.8 | Переписать `bootstrap/container.go` для новой структуры |
| 1.9 | Обновить `main.go` |
| 1.10 | Запустить `wails dev` — убедиться, что бэкенд компилируется и Wails-биндинги генерируются |

**Критерий готовности:** `wails dev` запускается, генерируются новые JS-биндинги для 4 обработчиков.

### Фаза 2: Ядро фронтенда

**Цель:** Ядро фронтенда (редактор, плагины, workspace) работает с новой структурой.

| Шаг | Действие |
|-----|----------|
| 2.1 | Создать директорию `kernel/` с новой структурой |
| 2.2 | Переместить редактор → `kernel/editor/` (разбить по функциям: core, extensions, serialization, auto-save, image-handling, sessions, toc, ui) |
| 2.3 | Переместить систему плагинов → `kernel/plugin-system/` (обновить до API v5) |
| 2.4 | Переместить менеджер workspace → `kernel/workspace/` |
| 2.5 | Переместить навигацию → `kernel/navigation/` |
| 2.6 | Создать дизайн-токены → `shared/design-tokens/` |
| 2.7 | Редизайн всех UI-компонентов → `shared/ui/` |
| 2.8 | Добавить линт-контроль (stylelint + ESLint) |

**Критерий готовности:** Ядро компилируется, редактор работает, плагины загружаются (пока без встроенных плагинов).

**Критерий готовности:** Все проверки пройдены, приложение полностью функционально.

## 3. Оценка рисков

| Риск | Влияние | Митигация |
|------|---------|-----------|
| Поломка API бэкенда | Высокое | Сгенерировать новые Wails-биндинги, протестировать все 4 обработчика до работы с фронтендом |
| Несовместимость API v5 плагинов | Высокое | Предоставить миграционный гайд; плагины v4 показывают предупреждение о депрекации |
| Баги при разделении функций редактора | Среднее | Тестировать каждую функцию редактора независимо после миграции |
| Миграция дизайн-токенов | Среднее | Визуальное регрессионное тестирование со скриншотами Playwright |
| Сложность big bang миграции | Высокое | Выполнить все фазы в одной ветке; не мержить инкрементально |
| Потеря функциональности | Высокое | Создать чек-лист всех текущих функций; проверить каждую после миграции |
