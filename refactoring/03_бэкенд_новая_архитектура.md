## 2. Что изменяется

| Старое расположение | Новое расположение | Действие |
|---------------------|-------------------|----------|
| `core/file/` | `backend/domain/file/` | Переместить без изменений |
| `core/volt/` | **УДАЛЕНО** | Сущность Volt перемещается в плагин фронтенда |
| `core/note/` | **УДАЛЕНО** | Note — это просто FileEntry с расширением `.md` — отдельный домен не нужен |
| `core/search/` | **УДАЛЕНО** | Поиск перемещается в плагин фронтенда |
| `core/settings/` | **РАЗДЕЛЕНО** | Сервис локализации → фронтенд; KV-хранилище → `backend/domain/storage/` |
| `core/plugin/` | **УДАЛЕНО** | Управление плагинами полностью перемещается на фронтенд |
| `commands/file/` | `backend/application/file/` | Команды становятся методами `FileApplicationService` |
| `commands/volt/` | **УДАЛЕНО** | Перемещается в плагин управления vault фронтенда |
| `commands/note/` | **УДАЛЕНО** | Создание заметки — это `file.Create()` с расширением `.md` |
| `commands/search/` | **УДАЛЕНО** | Перемещается в плагин поиска фронтенда |
| `commands/plugin/` | **УДАЛЕНО** | Полностью перемещается на фронтенд |
| `commands/settings/` | **УДАЛЕНО** | Перемещается в функцию настроек фронтенда |
| `commands/system/` | **РАЗДЕЛЕНО** | Диалоги → `backend/application/dialog/`; Процессы → `backend/application/process/`; Изображения → **УДАЛЕНО** (фронтенд); Link preview → **УДАЛЕНО** (фронтенд); Копирование ассетов → **УДАЛЕНО** (сервис фронтенда) |
| `infrastructure/filesystem/` | `backend/infrastructure/filesystem/` | Переместить без изменений |
| `infrastructure/persistence/` | `backend/infrastructure/storage/` | Переместить, переименовать |
| `infrastructure/runtime/wails/` | `backend/infrastructure/wails/` | Переместить, переименовать |
| `interfaces/wailshandler/` | **РАЗДЕЛЕНО** | 9 обработчиков → 4 обработчика (file, process, dialog, storage) |

## 3. Поверхность Wails API бэкенда (после)

| Обработчик | Методы | Назначение |
|-----------|--------|-----------|
| `FileHandler` | `Read(path)`, `Write(path, content)`, `ListTree(root)`, `CreateFile(path)`, `CreateDirectory(path)`, `Delete(path)`, `Rename(oldPath, newPath)` | CRUD файловой системы |
| `ProcessHandler` | `Start(config)`, `Cancel(id)` | Управление ОС-процессами |
| `DialogHandler` | `SelectDirectory()`, `PickFiles(accept, multiple)`, `PickImage()` | Системные диалоги |
| `StorageHandler` | `Get(namespace, key)`, `Set(namespace, key, value)`, `Delete(namespace, key)`, `List(namespace)` | KV-персистентность |

**Всего обработчиков: 4** (было 9)

## 4. Детализация слоёв

### 4.1 Domain-слой

Domain-слой содержит **только**:
- Сущности (структуры данных с бизнес-инвариантами)
- Интерфейсы репозиториев (контракты для инфраструктуры)
- Доменные ошибки (специфичные для домена)

Domain-слой **не знает** ни о чём внешнем:
- Не импортирует `database/sql`, `os`, `wails`
- Не содержит HTTP/Wails обработчики
- Не содержит логику use-case'ов

### 4.2 Application-слой

Application-слой содержит:
- **Application-сервисы** — оркестрируют вызовы репозиториев, применяют бизнес-правила
- **DTO** — структуры запросов и ответов для каждого use-case'а
- **Ошибки уровня приложения** — ошибки валидации, авторизации и т.д.

Application-слой **не знает** об инфраструктуре:
- Не импортирует `os`, `wails`, конкретные реализации репозиториев
- Работает только с интерфейсами из domain-слоя

### 4.3 Infrastructure-слой

Infrastructure-слой содержит:
- **Конкретные реализации репозиториев** — `FileRepository` (os), `ProcessRepository` (os/exec), `DialogRepository` (wails runtime)
- **KV-хранилища** — `VoltStore`, `SettingsStore` (JSON-файлы в `~/.volt/`)
- **Wails runtime** — мост для событий, логирования, диалогов

### 4.4 Interfaces-слой

Interfaces-слой содержит:
- **Wails-обработчики** — принимают вызовы от фронтенда, делегируют application-сервисам, возвращают результаты
- **Lifecycle** — хуки `Startup`, `DomReady`, `BeforeClose`
- **VaultAssetServer** — HTTP-обработчик для раздачи ассетов из vault'ов

Каждый обработчик:
- Не содержит бизнес-логики
- Только вызывает application-сервисы
- Не обрабатывает локализацию ошибок (это делает фронтенд)
